春招感想

标题没叫春招总结，是因为总结都是拿到offer了，我是 春招 0 offer ，现在已经准备结束春招，备战秋招了。

突然明白为什么强点的大公司都偏向于早点开始招聘，除了想要抢人才，也许还有一个原因，就是学生一开始打了鸡血般的热情会在逐渐的笔试面试中消耗，直到最后放弃 or 拿到offer，早一点开始面，学生面试的热情就越高哇，投的人自然也多。

这一个半月笔面下来，全军覆没，要么挂了，要么被扔池子里泡着，足以说明我的基础知识不够扎实，以及缺少项目经历这个硬伤了。

我清楚秋招时没有实习经历会很难，但是现在 继续面下去增长面试经验 和 静下心来系统学习 ，这两个选择权衡下来，对我来说，选择后者会更有利，理由我在别的地方已经总结过了，这里就不再说了。

下面是我对春招复习过程出现的问题的反思：

1. 过于针对面试复习

每次准备面试，总会在前一天或者前两天，把一些常问的东西过一遍，因此，长时间下来，因为笔试面试的频率较高，所以这段时间总是在应试，先不说背离学技术的初衷，就这样一直短时间内应试，面试时一旦深入去问，或者需要横向比较时，必然会挂。

2. 没有计划

寒假时有看一些面经，看面经的时候发现，%#$%&*，我怎么什么都不会，发现不会的然后就去学，学的零零散散。正式春招开始后也是，接到面试通知，就开始搜面经，然后针对面经看看、背背，这就是所谓的 低级应试了。

真正学习能力强的人，一定有一个规划，在什么时间要通过怎样的手段达到怎样的目标，时间、手段、目标都要恰当。

在网上看到一个复旦研究生小姐姐的春招过程，瞬间明白什么是学习能力强的人。她和我的基础知识掌握程度看起来差不太多，项目经历也是，但是我们两个看起来最大的区别就是学习能力，学习能力很大一方面会体现在对自己的规划上，她的春招过程比我高效高太多了。基础方面：有计划地学习并一直坚持下去，并不受面试结果的影响；项目方面：她在网上找了一个项目并且半个月把这个项目做完且基本吃透了，这个我是很难做到，而且一开始我也没这个胆量，总是认为完成一个项目要持续很久，但是为了面试中展示自己的学习能力，只花少量时间去高效完成一个东西又何尝不可呢？

接下来我必须做一个详细的计划了。

3. 没有项目

这个很尴尬，项目没有是硬伤，死扛基础扛不过。就算基础好，但是有项目的人基础也不一定差，所以就被比下去了。

同时，我要做的是工程师啊工程师！不是走算法方向，走开发自然要有工程思维，工程思维怎么来，多做项目就知道了，同时，没有项目经验，哪个公司带你这个新手都很麻烦。大公司嘴上说是主要看基础，这个没错，基础当然很重要，但是项目也必须要有，除非，能达到精通xxx的程度，那自然让你学一个框架，基本稍微看看就知道怎么回事，反正我目前是达不到。

再有就是，有项目的话，一方面是体现自身的学习和动手能力，另一方面，以项目为驱动来学习，效果会比泛泛地学更好，同时，对于语言和一些基础知识的理解还是需要应用在项目中才能理解到位。

4. 目光短浅

这个是我在和前辈聊过之后才感觉出来的，具体不方便写出来了，主要是涉及到公司前景，职位导向，工作城市，未来发展等等。现在感觉自己真的是一个未出茅庐的小孩子，人生路漫漫，有很多需要学习和思考的地方，不急，慢慢来。

5. 心态

这个其实不算是春招出现的问题，不过因为心态这个东西也很重要，就放在这里一起总结一下。

很多人都说找工作心态占了相当重要的一部分，我认为，心态这种东西，是大多数人都应学会调整的。心态问题其实就是抗压能力，对于我一个从山东数十万高考考生中走出来的学生，抗压对我来说不是大问题，我有一套自己的调整方案，不过在这方面我仍然需要学习那些能做到：一边面试还能一边系统且有计划地学习的人，这种人的学习能力不一般，心态也不一般。

还有就是，不要和别人比。总有一些 offer 收割机，但是这些人是多年的沉淀 or 智商很高 or 兴趣浓厚，总之就是有自己的强大的优势在的。这些大佬吧，也不能说是有意的，但是对 0 offer 的人来说，他们的每次收割和炫耀，对自己难免会有些打击，但是做好自己就好了，我不求能收割多个 offer，只求一个能让自己称心如意舒舒服服的把自己卖了的公司。

6. 算法薄弱

这个，怪自己菜咯，做题少，没感觉。办法只有一个，继续做题，只是怎么做能更高效需要思考一下。

那么，针对以上几个主要问题，我需要找到相应解决方案，并能执行下去，这一步，比较难，而且有可能踩不少坑，但是无法避免，只能硬着头皮上。

1. 系统看书

在网上已经看过不少人讨论过看博客、看面经、看书、看视频学习方式的有效程度了，大家各执己见，这没错，因为每个人适合的方式不同，而我，在踩了不少坑后，目前我认为，我适合先粗看博客，再看书（选取部分章节）进行系统学习。

这样会慢，但是全程不看书，只看博客 or 视频，对我来说很难理解东西理解到一定程度，所以，我需要做一个看什么书、怎么看、什么时候看完的一个规划。这个规划如下：

  • 需要看的书：
    • Core Java 和 TIJ（跟着自己做题遇到模糊的去看，然后总结写笔记）

    • 深入理解Java虚拟机（全都要看，目前只看了面试常问的一部分）
    • 高性能MySQL（部分章节：事务、Innodb和myisam引擎、索引、查询性能优化等等）
    • Spring in action & Spring 3.x企业应用开发实战（框架我懂的少，这个和博客一起看）

2. 项目

项目这个没啥可说的，学习，做，理解，思考，优化。然后等着秋招面试被虐几次就知道项目怎么讲了。

对项目用到的技术栈，针对性的学习，Spring 必须懂，Mybatis、redis都有很多可讲的地方，如果最后有时间，看源码，看框架用设计模式用得是多么的炉火纯青。

最后把项目架构画出来，再对做项目过程遇到的难题进行总结。

项目需要在五月底之前完成，对于优化和一些细节的处理可以在六七月份完成。
